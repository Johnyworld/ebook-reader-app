# 테스트 케이스 문서

## 실행 환경

- Android 에뮬레이터 API 34 (Android 14)
- `./gradlew connectedDebugAndroidTest` 로 전체 실행

---

## 1. 홈 화면

### TC-1.1 책 목록이 정상적으로 표시된다

| 항목 | 내용 |
|---|---|
| 사전 조건 | 기기에 EPUB 파일이 1개 이상 존재 |
| 동작 | 앱을 실행한다 |
| 기대 결과 | 홈 화면에 책 제목 목록이 표시된다 |

### TC-1.2 책을 탭하면 리더 화면으로 이동한다

| 항목 | 내용 |
|---|---|
| 사전 조건 | 홈 화면에 책 목록이 표시된 상태 |
| 동작 | 책 아이템을 탭한다 |
| 기대 결과 | 리더 화면이 표시된다 |

### TC-1.3 책이 없으면 "비어있음" 메시지가 표시된다

| 항목 | 내용 |
|---|---|
| 사전 조건 | 기기에 지원되는 도서 파일이 없음 |
| 동작 | 앱을 실행한다 |
| 기대 결과 | "비어있음" 텍스트가 화면에 표시된다 |

### TC-1.4 책 목록에 저자 정보가 표시된다

| 항목 | 내용 |
|---|---|
| 사전 조건 | 메타데이터에 저자 정보가 포함된 도서가 존재 |
| 동작 | 앱을 실행한다 |
| 기대 결과 | 각 책 아이템에 저자명이 함께 표시된다 |

### TC-1.5 스플래시 화면 후 책 목록으로 전환된다

| 항목 | 내용 |
|---|---|
| 사전 조건 | 없음 |
| 동작 | 앱을 실행한다 |
| 기대 결과 | "ebook-reader" 스플래시가 잠시 표시된 후 사라지고, 책 목록이 나타난다 |

## 2. 검색

### TC-2.1 검색어를 입력하면 일치하는 책만 표시된다

| 항목 | 내용 |
|---|---|
| 사전 조건 | 홈 화면에 책 2권 이상 표시된 상태 |
| 동작 | 1. 검색 아이콘을 탭한다<br>2. 검색어를 입력한다 (예: 책 제목의 일부) |
| 기대 결과 | 검색어와 일치하는 책만 목록에 표시되고, 나머지는 사라진다 |

## 3. 정렬

### TC-3.1 정렬 기준을 변경하면 목록 순서가 바뀐다

| 항목 | 내용 |
|---|---|
| 사전 조건 | 홈 화면에 책 2권 이상 표시된 상태 |
| 동작 | 1. 정렬 드롭다운을 탭한다<br>2. "작가순"을 선택한다 |
| 기대 결과 | 책 목록이 저자명 가나다순으로 정렬된다 |

## 4. 즐겨찾기

### TC-4.1 즐겨찾기에 추가하고 필터로 확인할 수 있다

| 항목 | 내용 |
|---|---|
| 사전 조건 | 홈 화면에 책이 표시된 상태 |
| 동작 | 1. 책의 더보기(⋮) 메뉴를 탭한다<br>2. "즐겨찾기"를 탭한다<br>3. 필터 드롭다운에서 "즐겨찾기"를 선택한다 |
| 기대 결과 | 즐겨찾기한 책만 목록에 표시된다 |

## 5. EPUB 뷰어 — 목차 네비게이션

### TC-5.1 목차에서 챕터를 선택한 후 이전 페이지로 이동하면 페이지 번호가 1 감소한다

| 항목 | 내용 |
|---|---|
| 사전 조건 | EPUB 파일이 열려있고 콘텐츠가 렌더링된 상태 |
| 동작 | 1. 화면을 탭하여 메뉴를 연다<br>2. 목차 버튼을 탭한다<br>3. 중간 챕터(예: 제3장)를 탭한다<br>4. 메뉴를 열어 현재 페이지 번호(A)를 확인한다<br>5. 메뉴를 닫고 화면 왼쪽을 탭하여 이전 페이지로 이동한다<br>6. 메뉴를 열어 현재 페이지 번호(B)를 확인한다 |
| 기대 결과 | B + 1 = A (이전 페이지로 정확히 1페이지 이동됨) |
