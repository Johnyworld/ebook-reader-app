# 다국어(i18n) 지원 설계

## 목적

앱을 글로벌 출시하기 위해 다국어를 지원한다.

## 지원 언어

| 언어 | 로케일 코드 | 비고 |
|------|------------|------|
| English | `en` | 기본 언어 (fallback) |
| 한국어 | `ko` | |
| 日本語 | `ja` | |
| 中文 | `zh` | 간체 |
| Español | `es` | 스페인어 |

## 번역 방식

- AI 번역(ChatGPT 등)으로 초안 생성 후 검수

## 적용 범위

- 앱 UI 텍스트 (메뉴, 버튼, 안내 문구 등)
- 날짜/숫자 로케일 포맷
- 책 뷰어 내부 콘텐츠(EPUB/PDF 본문)는 대상이 아님

---

## 1. 리소스 구조

```
app/src/main/res/
├── values/           # 기본 언어 (영어)
│   └── strings.xml
├── values-ko/        # 한국어
│   └── strings.xml
├── values-ja/        # 일본어
│   └── strings.xml
├── values-zh/        # 중국어 (간체)
│   └── strings.xml
└── values-es/        # 스페인어
    └── strings.xml
```

- `values/strings.xml`에 영어를 배치하여 매칭되는 로케일이 없을 때 fallback으로 사용
- 현재 하드코딩된 한글 문자열을 모두 키로 추출하여 `values-ko/strings.xml`에 배치
- 영어 번역을 `values/strings.xml`에 작성
- 문자열 키 네이밍 컨벤션: `screen_element_description` (예: `all_books_search_hint`, `reader_settings_font_size`)

## 2. Compose 코드 변경 패턴

하드코딩된 문자열을 `stringResource()`로 교체한다.

```kotlin
// Before
Text("파일 검색 중...")

// After
Text(stringResource(R.string.all_books_loading))
```

- Compose 함수 내부: `stringResource(R.string.key)`
- Compose 외부(ViewModel 등): `Context.getString(R.string.key)`
- 포맷 문자열: `stringResource(R.string.key, arg1, arg2)`

## 3. 앱 내 언어 선택 기능

### Android 13+ (API 33+)

시스템 설정의 Per-App Language를 자동 지원한다. `AndroidManifest.xml`에 `android:localeConfig`를 선언한다.

```xml
<!-- res/xml/locale_config.xml -->
<locale-config xmlns:android="http://schemas.android.com/apk/res/android">
    <locale android:name="en"/>
    <locale android:name="ko"/>
    <locale android:name="ja"/>
    <locale android:name="zh"/>
    <locale android:name="es"/>
</locale-config>
```

### Android 12 이하

`AppCompatDelegate.setApplicationLocales()`로 앱 내에서 언어를 변경한다. `AppCompatDelegate`가 내부적으로 persist하므로 별도 저장 로직은 불필요하다.

### UI

기존 설정 화면에 "언어(Language)" 항목을 추가한다. 선택지는 다음과 같다:

- English
- 한국어
- 日本語
- 中文
- Español

## 4. 날짜/숫자 로케일 처리

- **날짜 포맷**: `DateTimeFormatter.ofLocalizedDate()`를 사용하여 현재 로케일에 맞게 자동 포맷
  - 영어: `Apr 18, 2026`
  - 한국어: `2026. 4. 18.`
  - 일본어: `2026/04/18`
- **숫자 포맷**: `NumberFormat.getInstance()`로 로케일 기반 자동 처리
- 현재 코드에서 날짜/숫자를 직접 포맷하는 부분이 있으면 로케일 인식 API로 교체
- 별도 라이브러리 추가 없이 Java/Kotlin 표준 API 사용
